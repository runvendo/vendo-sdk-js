# Vendo Spec B Phase B4 — TypeScript @vendo/sdk 0.1

> **For agentic workers:** REQUIRED SUB-SKILL: `superpowers:subagent-driven-development`. Steps use checkbox tracking.

**Goal:** Grow `@vendodev/sdk` from a single `getCredential()` helper into the full Vendo SDK surface, mirroring the Python SDK 1:1. Ship as v0.1.0 (alpha for v1.0). Async-only (TS doesn't need a sync twin — async is idiomatic).

**Architecture:**
- Public surface mirrors Python `Vendo` / `vendo.aio`:
  - `new Vendo({ apiKey?, baseUrl?, ... })` class with module-level convenience.
  - `client.token(slug)`, `client.tokens(slugs)`, `client.invalidate(slug)`.
  - `client.connections.list/get`, `client.integrations.list/get`, `client.billing.balance/spendCaps/usage`.
  - `client.connectUrl(slug, { returnTo })`.
  - `client.forUser(jwt)` — SaaS multi-tenant.
  - `vendo.errors` — typed error hierarchy (VendoError + 9 subclasses).
  - `vendo.testing` — `MockClient`, `fakeConnection()`.
  - `vendo.reconciler` — `bootstrap()`, `start()` (Node-only; no-op in browser).
- Backwards compat: keep `getCredential()`, `CredentialResponse`, `VendoSdkError` exported. New errors are additive.
- Browser bundle: `@vendodev/sdk/browser` exposes `connectUrl()` + a Web Component (deferred to B5 polish — not required for B4 alpha).
- Internal HTTP adapter: native `fetch` (Node 18+ has it built-in), retry policy, auto-`Idempotency-Key`, `Vendo-Error-Code` mapping.

**Tech stack:** TypeScript 5.6, tsup for build, Vitest for tests, native `fetch`. No new runtime deps.

**Naming note:** Spec calls for `@vendo/sdk` on npm. The repo currently publishes as `@vendodev/sdk` (the `@vendo` scope on npm may not be owned). For B4, keep `@vendodev/sdk`. B6 will reconcile naming if `@vendo` becomes available.

## Task 2: HTTP adapter (retry + idempotency)

**Files:** `src/_http.ts`, `src/_http.test.ts`

Mirror Python `_http.py`:
- `RetryPolicy { maxAttempts, backoffBaseMs, backoffMaxMs, retryableStatusCodes }`.
- `HttpAdapter` constructor: `apiKey, baseUrl, retry, apiVersion, timeout, fetch?`.
- `get/post/put/delete` methods; POST/PUT/DELETE auto-add `Idempotency-Key: <crypto.randomUUID()>` (override via opts).
- Retry on 5xx + network errors; backoff with jitter via `await new Promise(r => setTimeout(r, ms))`.
- Default headers: `User-Agent: @vendodev/sdk/0.1.0`, `Accept: application/json`, `Authorization: Bearer <key>`, `Vendo-API-Version: 2026-05-02`.
- Returns parsed JSON or throws `fromResponse(...)` on non-2xx.

Tests: 6 — same as Python (3 retry + 3 idempotency). Mock `fetch` via `vi.fn()`.

- [ ] TDD steps + commit `feat(http): adapter with retry + auto idempotency-key`.

## Task 3: Vendo class + auth

**Files:** `src/_client.ts`, `src/_client.test.ts`

```ts
export interface VendoOptions {
  apiKey?: string;
  baseUrl?: string;
  apiVersion?: string;
  retryPolicy?: RetryPolicy;
  timeout?: number;
}

export class Vendo {
  apiKey: string;
  baseUrl: string;
  readonly _http: HttpAdapter;
  readonly connections: ConnectionsAPI;
  readonly integrations: IntegrationsAPI;
  readonly billing: BillingAPI;
  private _userJwt?: string;

  constructor(opts: VendoOptions = {}) {
    this.apiKey = opts.apiKey ?? process.env.VENDO_API_KEY ?? "";
    if (!this.apiKey) throw new AuthError("VENDO_API_KEY not set", { code: "app_unknown" });
    this.baseUrl = opts.baseUrl ?? process.env.VENDO_BASE_URL ?? "https://vendo.run/api";
    this._http = new HttpAdapter({ ... });
    this.connections = new ConnectionsAPI(this._http);
    this.integrations = new IntegrationsAPI(this._http);
    this.billing = new BillingAPI(this._http);
  }

  forUser(userJwt: string): Vendo {
    const v = new Vendo({ apiKey: this.apiKey, baseUrl: this.baseUrl, ... });
    v._userJwt = userJwt;
    return v;
  }

  async token(slug: string): Promise<string> { ... }
  async tokens(slugs: string[]): Promise<Record<string, string | null>> { ... }
  async invalidate(slug: string): Promise<void> { ... }
  connectUrl(slug: string, opts?: { returnTo?: string; state?: string }): string {
    return connectUrl(slug, { apiKey: this.apiKey, baseUrl: this.baseUrl.replace(/\/api$/, ""), ...opts });
  }
}
```

Token cache: per-slug `Map<string, { token: string; expiresAt: number }>`, TTL = `expires_at - now - 60s`. Browser-safe (no Node-only globals in this file).

Tests: 5 — constructor variants, env fallback, AuthError on missing key, `forUser` returns new instance, `connectUrl` works.

- [ ] TDD steps + commit `feat(vendo): Vendo class with token cache`.

Same shape as Python:

```ts
export interface ReconcilerOptions {
  envFile: string;
  mapping: () => Promise<Record<string, string>> | Record<string, string>;
  onChange?: "restart" | "reload-hook" | ((diff: { added: string[]; removed: string[]; changed: string[] }) => void);
  reloadUrl?: string;
}

export async function bootstrap(opts: ReconcilerOptions): Promise<void> {
  // Validate env: throw if not running in Node (typeof process === "undefined").
  // Import fs/promises dynamically so browser bundle doesn't pull it in.
  const { writeFile, rename, readFile, stat } = await import("node:fs/promises");
  ...
}

export async function start(opts: ReconcilerOptions): Promise<Reconciler> {
  await bootstrap(opts);
  const stopped = { v: false };
  const interval = setInterval(async () => {
    if (stopped.v) return;
    try {
      const newEnv = await opts.mapping();
      // diff + write + onChange (per python pattern)
    } catch {} // swallow
  }, 30_000);
  return {
    stop() { stopped.v = true; clearInterval(interval); }
  };
}
```

Tests: 4 — bootstrap writes atomically, skips unchanged, start returns object with `stop()`, `_applyDiffAndCall` invokes callable on diff. Use `os.tmpdir()` + cleanup.
